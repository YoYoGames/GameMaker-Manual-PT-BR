Opções de jogo Ubuntu

Esta seção descreve as diferentes opções disponíveis para você que controlam como seus projetos de jogos Ubuntu (Linux) serão compilados. As diferentes seções são:

Geral

Esta aba é onde você define o nome de seu jogo como ele deve ser exibido durante a execução, e também pode fornecer um e-mail de contato e uma descrição (curta e longa), assim como uma URL para a página inicial do jogo ou do desenvolvedor.

Aqui você também tem a opção de habilitar o Steam em seu jogo. Se você ativar esta opção, certifique-se de ter o Steamworks SDK instalado (veja aqui), e que você tenha adicionado o Steam app ID nas Opções Gerais para o jogo.

Finalmente, você tem a opção de Desativar o sistema de arquivo sandbox. Ao verificar isso, você desabilitará o sandboxing GameMaker para salvar e carregar arquivos, permitindo que seus jogos acessem arquivos de qualquer lugar do sistema que esteja rodando o jogo. Esta opção é desmarcada por padrão, pois recomendamos que você mantenha o sandbox do GameMaker para arquivos em todos os sistemas e, quando marcada, algumas áreas de salvamento/carregamento podem ainda não estar disponíveis, dependendo das permissões de nível de sistema operacional. Para maiores informações sobre a caixa de areia, consulte as páginas do Sistema de Arquivos GameMaker.

Gráficos

As opções gráficas são aquelas que você deve configurar para determinar como seu jogo utilizará a placa gráfica de sua máquina alvo Ubuntu. As seguintes opções estão incluídas para que você possa modificar:

  • Iniciar tela cheia: Se isto for verificado, o jogo começará no modo de tela cheia, caso contrário, começará em tela cheia. Isto é desligado por padrão.
  • Permitir a troca de tela cheia: Com isso, o usuário pode switch de tela cheia para janela e voltar usando os atalhos padrão Ubuntu OS shortcuts. Isto está desativado por padrão.
  • Interpolar as cores entre os pixels: Ativa a interpolação linear, que basicamente "suaviza" os pixels. Para gráficos de pixels nítidos, ele deve estar desligado, mas se você tiver misturas alfa agradáveis e gráficos alisados é melhor deixá-lo ligado.
  • Cursor de exibição: Quando esta opção estiver ativada, o cursor padrão do SO será mostrado em seu jogo. Se esta estiver desabilitada, então enquanto o mouse estiver sobre a janela do jogo, não haverá cursor visível a menos que você tenha criado um como parte do projeto do jogo (isto está ligado por padrão).
  • Use a sincronização para evitar rasgos: Esta sincronização ativa ou desativa a sincronização v-sync. Note que se você tiver um jogo com velocidade room de 120 e o jogador tiver um monitor com uma taxa de atualização de 60, ligar esta opção bloqueará sua velocidade de jogo para 60 também. Esta opção está desligada por padrão.
  • Permitir o redimensionamento da janela: A verificação disto permite ao usuário mudar o tamanho da janela do jogo (a opção Janela sem Fronteiras precisa estar desligada para que isto funcione). Esta opção está desativada por padrão.
  • Escalonamento: Aqui você pode optar por manter a relação de aspecto (assim um 4:3 room será "letter boxed" em um 16:9) ou escalar completamente (esticando a imagem para caber na tela inteira).

    ADVERTÊNCIA! A desativação da superfície de aplicação desativará todas as opções de escalonamento definidas no Windows Game Options até que tenha sido ligada novamente. Veja a superfície de aplicação para mais detalhes.

Finalmente, há a opção de definir o tamanho da página de textura. O tamanho padrão (e o mais compatível) é 2048x2048, mas você pode escolher entre 256x256 até 8192x8192. Há também um botão marcado Pré-visualização que vai gerar as páginas texture para esta plataforma e depois abrir uma janela para que você possa ver como elas são. Isto pode ser muito útil se você desejar ver como as páginas texture estão estruturadas e para evitar ter páginas texture maiores (ou menores) do que o necessário. Para mais informações sobre as páginas texture, por favor, veja aqui.

NOTA: Esteja ciente de que quanto maior o tamanho da página texture, menos compatível seu jogo estará nos PCs com especificações mais baixas.

Imagens

Aqui você pode adicionar uma tela Splash para seu jogo, e definir o ícone do jogo que será usado. O ícone deve ser uma imagem de 64x64 pixels no formato .png, e a tela splash deve ter pelo menos o mesmo tamanho que a primeira room em seu jogo (ou sua porta de visualização) e ser no formato .png também. A tela splash será mostrada enquanto o jogo carrega, e pode ser habilitada ou desabilitada verificando a opção Use Splash Screen (que está desligada por padrão).

Vale notar que o GameMaker possui uma ferramenta Gerador de Imagens de Projeto que pode ser usada para criar automaticamente todas as imagens necessárias para todas as diferentes plataformas alvo para as quais seu jogo está sendo compilado. Se você usar esta ferramenta, você deve revisar as imagens criadas para garantir que elas sejam o que você precisa.
