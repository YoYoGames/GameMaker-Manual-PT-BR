Opções de jogos do Windows

Esta seção descreve as diferentes opções disponíveis para você que controlam como seus projetos de jogo Windows serão compilados. As diferentes seções são:

Geral

A página Geral para Windows jogos é onde você deve fornecer as informações necessárias para Windows para exibir quando seu jogo tiver sido instalado e estiver funcionando. Estas informações vão na seção Informações sobre o produto, e devem ser preenchidas completamente.

A seção Opções desta janela permite que você escolha para onde o pacote do jogo será instalado, seja para as pastas "%localappdata%" ou "%appdata%", assim como se deve definir Copy exe para a pasta de saída. Em geral, você nunca deve precisar desta opção, mas ela é fornecida para aqueles usuários que possuem várias DLL's que se comunicam entre si e com seu jogo, e quando ativada, ela manterá o jogo EXE na pasta de saída ao testar, em vez de colocá-lo na pasta Temp .

A seguir você pode definir a Margem de Sono do Windows. Esta opção está relacionada à redução da gagueira ao executar seu jogo em sistemas específicos. Basicamente, se seu jogo estiver rodando mais rápido que seu room speed GameMaker "dormirá" pelo tempo restante, mas este sono pode ser bastante impreciso e muitas vezes você pode parar de dormir por mais tempo do que o necessário, fazendo com que seu quadro demore mais, o que causa gagueira. Para contornar isso, podemos dormir por menos tempo, depois sentar em um loop apertado pelo restante do tempo para torná-lo mais preciso - embora o problema de sentar em um loop seja que isso faz com que o uso da CPU aumente, o que, por sua vez, pode fazer com que a temperatura de sua CPU aumente e seu ventilador funcione mais rápido. Por padrão, este valor é ajustado para 10, e em 99,99% dos casos você estará bem com isto, mas para máquinas de baixa potência ou para máquinas com muitos processos de fundo rodando isto pode não ser uma solução ideal (ou para máquinas de alta potência também) e um valor de 1 ou 15 ou algo assim pode ser necessário. Observe que esta é uma configuração muito específica do sistema e o que funciona em sua máquina construída pode não ser apropriado para outro usuário e, como tal, em caso de dúvida, deixá-la configurada para 10.

NOTA: Além da Margem de Sono, você pode controlar ainda mais seu desempenho e consumo de energia em Windows, ajustando a resolução do agendador de fios em runtime.

A próxima opção é Habilitar o vapor para seu jogo. Se você ativar esta opção, certifique-se de ter o Steamworks SDK instalado (veja aqui), e que você tenha adicionado o Steam App ID nas Opções Gerais para o jogo. Observe que se você estiver executando o GameMaker em Windows e a partir da versão Steam do IDE e estiver desenvolvendo jogos para Steam, então talvez você também precise verificar a opção Usar lançador alternativo para a aplicação Steam. O que esta opção faz é lançar seu jogo ao testar usando um programa lançador alternativo para evitar que ele seja sinalizado por Steam com o ID do aplicativo GameMaker em vez do ID do próprio aplicativo de seu jogo. Você só deve habilitar isto quando estiver desenvolvendo para Steam e se você encontrar problemas para lançar seu jogo.

A penúltima opção que você tem é a opção de desativar o sistema de arquivo sandbox. Ao verificar isso, você desabilitará o sandboxing GameMaker para salvar e carregar arquivos, permitindo que seus jogos tenham acesso a arquivos de qualquer lugar do sistema que esteja rodando o jogo. Esta opção é desmarcada por padrão, pois recomendamos que você mantenha o sandbox do GameMaker para arquivos em todos os sistemas e, quando marcada, algumas áreas de salvamento/carregamento podem ainda não estar disponíveis, dependendo das permissões de nível de sistema operacional. Para maiores informações sobre a caixa de areia, consulte as páginas do Sistema de Arquivos GameMaker.

Finalmente, você tem a opção de fazer com que o GameMaker crie um executável x64 para seus projetos Windows. Quando ativada, esta opção criará um executável de 64 bits para seu jogo quando ele for compilado, e recomendamos fortemente que todos os novos projetos utilizem esta opção. Este se tornará o formato padrão (e único) para os executáveis Windows no futuro a longo prazo, já que o executável de 32 bits estará sendo gradualmente eliminado. Note que, quando ativado, qualquer extension dlls sendo usada pelo projeto também precisará ser de 64 bits (veja aqui para mais informações).

Gráficos

As opções gráficas são aquelas que você deve configurar para determinar como seu jogo utilizará a placa gráfica de seu PC alvo Windows. As seguintes opções estão incluídas para que você possa modificar:

  • Cursor de exibição: Quando isto for verificado, o cursor normal das janelas será mostrado, e desmarcá-lo significa que nenhum cursor será mostrado a menos que você tenha criado um em seu código de jogo. Isto está ativado por padrão.
  • Iniciar tela cheia: Se isto for verificado, o jogo começará no modo de tela cheia, caso contrário, começará em tela cheia. Isto é desligado por padrão.
  • Permitir a troca em tela cheia: Com isto assinalado, o usuário pode switch de tela cheia para janela e voltar usando os atalhos padrão Windows. Isto está desativado por padrão.
  • Interpolar as cores entre os pixels: Ativa a interpolação linear, que basicamente "suaviza" os pixels. Para gráficos de pixels nítidos, ele deve estar desligado, mas se você tiver misturas alfa agradáveis e gráficos alisados é melhor deixá-lo ligado.
  • Use a sincronização para evitar rasgos: Esta sincronização ativa ou desativa a sincronização v-sync. Note que se você tiver um jogo com velocidade room de 120 e o jogador tiver um monitor com uma taxa de atualização de 60, ligar esta opção bloqueará sua velocidade de jogo para 60 também. Esta opção está desligada por padrão.
  • Permitir o redimensionamento da janela: A verificação disto permite ao usuário mudar o tamanho da janela do jogo (a opção Janela sem Fronteiras precisa estar desligada para que isto funcione). Esta opção está desativada por padrão.
  • Janela sem fronteiras: Ao verificar isso, seu jogo será executado com uma janela sem bordas, removendo os botões normais de minimizar, maximizar e fechar, assim como o título do jogo. Isto é desligado por padrão.
  • Escalonamento: Aqui você pode optar por manter a relação de aspecto (assim um 4:3 room será "letter boxed" em um 16:9) ou escalar completamente (esticando a imagem para caber na tela inteira).

    ADVERTÊNCIA! A desativação da superfície de aplicação desativará todas as opções de escalonamento definidas no Windows Game Options até que tenha sido ligada novamente. Veja a superfície de aplicação para mais detalhes.

Finalmente, há a opção de definir o tamanho da página de textura. O tamanho padrão (e o mais compatível) é 2048x2048, mas você pode escolher entre 256x256 até 8192x8192. Há também um botão marcado Pré-visualização que vai gerar as páginas texture para esta plataforma e depois abrir uma janela para que você possa ver como elas são. Isto pode ser muito útil se você desejar ver como as páginas texture estão estruturadas e para evitar ter páginas texture maiores (ou menores) do que o necessário. Para mais informações sobre as páginas texture, por favor, veja aqui.

NOTA: Esteja ciente de que quanto maior o tamanho da página texture, menos compatível seu jogo estará nos PCs com especificações mais baixas.

Imagens

A seção de imagens é onde você fornece as imagens que seu jogo requer. Para Windows, você precisa fornecer um arquivo de ícones (que deve estar no formato .ico ) e também uma tela de respingo (que pode ser no formato .png, .bmp, .jpg ou .gif ). A tela splash será mostrada enquanto o jogo carrega, e pode ser habilitada ou desabilitada verificando a opção Use Splash Screen (que está desligada por padrão).

Vale notar que o GameMaker possui uma ferramenta Gerador de Imagens de Projeto que pode ser usada para criar automaticamente todas as imagens necessárias para todas as diferentes plataformas alvo para as quais seu jogo está sendo compilado. Se você usar esta ferramenta, você deve revisar as imagens criadas para garantir que elas sejam o que você precisa.

Instalador

Os jogos Windows podem ser criados com um instalador e aqui você pode definir as várias configurações e gráficos que o GameMaker utilizará ao criá-lo. Antes de mais nada, temos as opções gráficas que definirão o visual do próprio instalador:

  • Acabado: O gráfico que aparece na página "Acabado" do instalador. Deve estar no formato *.bmp e medir 164x314px.
  • Cabeçalho: O gráfico que aparece enquanto o instalador está rodando. Deve estar no formato *.bmp e medir 150x57px.

Você também pode apontar para um arquivo específico do Contrato de Licença (um exemplo é fornecido com o GameMaker) e um arquivo personalizado NSI script (detalhes aqui) caso você precise de um, ou clique nos botões apropriados para retornar aos arquivos padrão fornecidos com o GameMaker.
