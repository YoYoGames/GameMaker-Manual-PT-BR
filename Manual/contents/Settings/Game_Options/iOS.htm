Opções de jogo iOS

Esta seção descreve as diferentes opções disponíveis para você que controlam como seus projetos de jogo iOS serão compilados. As diferentes seções são:

Geral

A primeira parte da seção Geral cobre os Build Settings para seu jogo. Para começar, defina as Orientações nas quais seu jogo pode ser executado para obter a melhor experiência de usuário. As opções disponíveis são:

  • Retrato
  • Retrato virado
  • Paisagem
  • Paisagem virada

Você pode então ir para a seção Suporte do Dispositivo, com as seguintes opções disponíveis para definir o tipo de dispositivo a ser visado:

  • iPhone ou iPod Touch: iPhone 6 e superior ou dispositivos iPod Touch
  • iPad: dispositivos iPad
  • Ambos: tanto iPhone/iPod quanto iPad

A segunda coisa a fazer é preencher as Informações do produto sobre seu jogo, incluindo seu Nome de exibição, seu ID do pacote e as Informações da versão. Note que o Bundle ID deve estar no formato correto de URL reversa para que a construção final do jogo funcione corretamente, EG: com.[Company].[GameName]. Aqui você também pode fornecer o Identificador de Equipe que você deseja usar para assinar o aplicativo final que o GameMaker cria para você. Defini-lo aqui substituirá o ID da Equipe que você forneceu nas Preferências iOS.

A seguir, você pode verificar ou desmarcar a opção de Deferir a Saída do Indicador de Casa. Quando isto for verificado, a barra de casa será sempre mostrada com um alfa baixo, e o primeiro deslize acorda a barra e o segundo deslize o leva para casa. Se a opção for desmarcada, então a barra não será mostrada no jogo, mas o primeiro deslize levará você para casa. A inadimplência não é verificada.

Finalmente, você tem a opção de gerar um Podfile e um Podfile.lock. Estes arquivos são necessários se você estiver adicionando CocoaPods a uma extensão iOS usando o Editor de Extensão. Os arquivos contêm macros específicas exigidas pelo GameMaker para configurar qualquer CocaoPods ao construir seu jogo. Clicando nos botões criará os arquivos com as macros pré-geradas, e também abrirá um explorador de arquivos na localização dos arquivos, caso você deseje editá-los manualmente. Há também um botão que o levará para o seguinte guia:

  • iOS e tvOS: usando cápsulas de cacau

Gráficos

Aqui você pode alterar os seguintes detalhes relacionados a como seu jogo será exibido, com as seguintes opções disponíveis:

  • Interpolar as cores entre os pixels: Ativa a interpolação linear, que basicamente "suaviza" os pixels. Para gráficos de pixels nítidos, deve ser desligado, mas se você tiver misturas alfa agradáveis e gráficos alisados, é melhor deixá-lo ligado. Isto está desativado por padrão.
  • Metade das texturas do iPad 1: O iPad 1 tem muito menos memória texture do que os dispositivos mais recentes, então você pode habilitar esta opção para reduzir o tamanho do textures pela metade se você quiser direcionar este dispositivo ou obter o máximo de compatibilidade.
  • Escala: Aqui você pode selecionar a escala da tela de desenho mantendo a relação de aspecto dentro da tela do dispositivo - adicionando "acolchoamento" ao redor das bordas para caber na tela - ou você pode selecionar ter a tela de desenho esticada para caber na tela do dispositivo.

Finalmente, há a opção de definir o tamanho da página de textura. O tamanho padrão (e o mais compatível) é 2048x2048, mas você pode escolher entre 256x256 até 8192x8192. Há também um botão marcado Preview que irá gerar as páginas texture para esta plataforma e depois abrir uma janela para que você possa ver como elas ficam. Isto pode ser muito útil se você desejar ver como as páginas texture estão estruturadas e para evitar ter páginas texture maiores (ou menores) do que o necessário. Para mais informações sobre as páginas texture, por favor, veja aqui.

NOTA: Esteja ciente de que quanto maior o tamanho da página texture, menos compatível seu jogo será com diferentes navegadores e dispositivos.

Ícones

Esta seção permite adicionar os vários ícones diferentes exigidos pelos vários dispositivos iPhone e iPad e as diferentes partes da App Store iOS. Estes ícones devem ser de autoria de imagens *.png de 24 bits no tamanho especificado para cada uma.

Vale notar que o GameMaker possui uma ferramenta Gerador de Imagens de Projeto que pode ser usada para criar automaticamente todas as imagens necessárias para todas as diferentes plataformas alvo para as quais seu jogo está sendo compilado. Se você usar esta ferramenta, você deve revisar as imagens criadas para garantir que elas sejam o que você precisa. A ferramenta gerará ícones assim como imagens (veja a seção abaixo).

NOTA Ao importar um projeto mais antigo, você poderá receber o seguinte aviso:Isto ocorre porque a versão atual iOS mudou o formato dos ícones necessários desde que o projeto foi carregado anteriormente. Você será solicitado a recriar todos os ícones, pois eles terão sido substituídos pelos ícones genéricos do GameMaker (você pode usar a ferramenta Project Image Generator acima mencionada para fazer isso rapidamente).

Imagens

Esta seção permite adicionar gráficos separados como Telas de Lançamento (a imagem que será mostrada brevemente enquanto seu aplicativo carrega) para cada uma das orientações do dispositivo, bem como a Cor da Borda que será usada como fundo para as telas de lançamento que não estão configuradas para preencher a tela inteira. Estas imagens devem ser autoradas como imagens individuais 24bit *.png sem transparências e em tamanho apropriado. Note que isto não é considerado uma tela de respingo e a Apple tem algumas diretrizes bastante rígidas sobre como isto deve ser configurado e utilizado. Veja a página do desenvolvedor em Diretrizes de Interface Humana para mais informações.

Você também pode definir como a tela de lançamento deve ser exibida a partir das seguintes opções:

  • Mantenha a relação de aspecto: Isto manterá a relação de aspecto da imagem em comparação com a tela do dispositivo, adicionando "acolchoamento" ao redor das bordas onde for necessário.
  • Cortar à medida: Isto manterá a imagem 1:1 com a tela do dispositivo e cortará (ou almofada) as bordas, conforme necessário.
  • Esticar: Isto esticará a imagem para preencher a tela do dispositivo (e pode distorcer a imagem mostrada).
