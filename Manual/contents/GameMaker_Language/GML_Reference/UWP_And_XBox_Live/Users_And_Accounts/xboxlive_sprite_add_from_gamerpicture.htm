xboxlive_sprite_add_from_gamerpicture

Com esta função, você pode obter a imagem do jogador para um determinado ponteiro de identificação do usuário. A função funciona de forma assíncrona, e acionará um evento assíncrono de imagem carregada para informar que a função está concluída (como nas chamadas HTML5/URL para sprite_add()). O evento Asynchronous Image Loaded terá um mapa DS armazenado na variável embutida async_load. O mapa contém as seguintes informações:

  • "nome do arquivo": O completo path para o arquivo que você solicitou.
  • "id": O ID do recurso que você carregou. Este será o mesmo que a variável à qual você atribuiu o recurso.
  • "status": Retorna um valor inferior a 0 para um erro.

NOTA: Quando você carrega dinamicamente um sprite em seu jogo em runtime, você deve lembrar de removê-lo novamente usando sprite_delete() quando não for mais necessário, caso contrário há o risco de um vazamento de memória que irá retardar e, eventualmente, quebrar seu jogo.

Sintaxe:

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig);

ArgumentoTipo Descrição
user_id The user ID (a pointer) to get the gamer picture for
imagesize Size in pixels of the sprite to be returned
xorig Indicate the x position of the origin in the sprite
yorig Indicate the y position of the origin in the sprite

Devoluções:

Async Request ID

Exemplo:

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
    var _uid = xboxlive_get_user(i);
    if (_uid != pointer_null)
    {
        global.UserName[_a] = xboxlive_gamedisplayname_for_user(_uid);
        global.UserScore[_a] = xboxlive_gamerscore_for_user(_uid);
        global.UserRep[_a] = xboxlive_reputation_for_user(_uid);
        global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture(_uid, 256, 0, 0);
        ++a;
    }
}

O código acima loops através dos usuários logados e armazena seus dados do jogador em várias arrays globais.
