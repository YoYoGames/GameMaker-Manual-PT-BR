buffer_create

Você usa esta função para alocar uma porção de memória como um buffer em seu jogo, com a função retornando o buffer id único que deve ser armazenado em uma variável e usado para todas as outras chamadas de função ao buffer. O buffer pode então ser usado para armazenar diferentes tipos de dados (especificado quando você escreve para o buffer usando o buffer_write() com as seguintes constantes sendo usadas para definir o tipo buffer:

Constante tipo tampão
Constante Descrição
buffer_fixed Um buffer de tamanho fixo.
buffer_grow Um buffer que irá "crescer" dinamicamente à medida que os dados forem sendo adicionados
buffer_wrap A buffer onde os dados serão "embrulhados". Quando os dados a serem adicionados atingirem o limite do tamanho buffer, o sobrescrito será colocado de volta no início do buffer, e a partir desse ponto continuará a ser escrito.
buffer_fast Especial "despojado" buffer que é extremamente rápido de ler/escrever. Só pode ser usado com os tipos de dados buffer_u8, e deve estar alinhado a 1 byte.
buffer_vbuffer Este tipo de buffer deve ser usado apenas como um vertex buffer.

Além do tipo buffer, você também terá que definir o alinhamento de bytes para o buffer. Este valor variará dependendo dos dados que você deseja armazenar no buffer, e na maioria dos casos um valor de 1 é perfeitamente bom. Entretanto, esteja ciente de que para algumas operações um alinhamento específico é essencial, e um alinhamento incorreto pode causar erros (para mais detalhes sobre alinhamento, veja Buffers). A seguir, um guia geral para mostrar quais valores são mais apropriados para cada tipo de dado (quando em dúvida, use um alinhamento de 1):

  • As cordas devem ser alinhadas a 1 byte.
  • Inteiros de 8 bits assinados ou não podem ser alinhados a qualquer valor, mas note que para um rápido buffer (veja buffer_write()) deve ser alinhado a 1.
  • Inteiros de 16 bits assinados ou não assinados devem ser alinhados a 2 bytes.
  • Os inteiros de 32 bits assinados ou não devem ser alinhados a 4 bytes
  • Flutuadores de até 16bits devem ser alinhados a 2 bytes. (Não suportado atualmente!)
  • Flutuadores de até 32bits devem ser alinhados a 4 bytes.
  • Flutuadores de até 64bits devem ser alinhados a 8 bytes.

NOTA: O alinhamento dos bytes pode ser muito importante, pois a escolha errada pode afetar negativamente o desempenho.

NOTA: É importante que você remova da memória qualquer recurso criado dinamicamente como este quando não precisar mais deles para evitar vazamentos de memória, então quando você terminar de usar o buffer que você criou você deve liberá-lo novamente usando buffer_delete().

Sintaxe:

buffer_create(size, type, alignment)

Argumento Tipo Descrição
size Real The size (in bytes) of the buffer.
type Constante tipo tampão The type of buffer to create (see the constants list above).
alignment Real The byte alignment for the buffer

Devoluções:

Buffer ID

Exemplo:

player_buffer = buffer_create(16384, buffer_fixed, 2);

O código acima aloca 16384 bytes de memória para um buffer e retorna o índice desse buffer, que é armazenado na variável "player_buffer", para uso futuro. O buffer está alinhado a um limite de dois bytes.
