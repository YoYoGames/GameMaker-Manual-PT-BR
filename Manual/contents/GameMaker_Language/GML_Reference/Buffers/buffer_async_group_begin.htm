buffer_async_group_begin

Esta função é chamada quando se deseja começar a salvar de múltiplos buffers para múltiplos arquivos (ou carregar múltiplos buffers). O nome do grupo é um string e será usado como o nome do diretório onde os arquivos são salvos, e deve ser usado como parte do arquivo path ao carregar os arquivos de volta para o IDE posteriormente (usando qualquer um dos buffer_load() funções). Esta função é apenas para uso com o buffer_save_async() e buffer_load_async() e você também deve terminar o grupo async chamando buffer_async_group_end() caso contrário, os arquivos não serão salvos/carregados.

Note que para as plataformas de console (como PS4 por exemplo), o nome do grupo será usado como a descrição do slot salvo, e usar esta função pode ajudá-lo a evitar que a IU mostre para cada arquivo que está sendo salvo. Observe também que ao utilizar o UWP, você pode navegar pelos jogos salvos entre as janelas das lojas Xbox nativas e Microsoft, criando um grupo que é o mesmo que um grupo ERA Xbox, por exemplo:

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

Sintaxe:

buffer_async_group_begin(groupname);

Argumento Tipo Descrição
groupname Cordão The name of the group (as a string).

Devoluções:

N/A

Exemplo:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

O código acima inicia um grupo buffer e depois o define para salvar 4 arquivos de forma assíncrona. A definição do grupo é então terminada (momento em que a gravação começará).
