gamepad_test_mapping

Esta função pode ser usada para definir o mapeamento do gamepad sobre os alvos que o permitem. Você fornece o índice "slot" para o gamepad definir, e depois o mapa string, que deveria ter sido criado usando o formato SDL com os seguintes campos:

<guid>,<description>,platform:<platform-name>,<bindings>

Os detalhes que devem ser incluídos em cada um desses campos são:

  • guid - um string de dígitos e letras que identificam de forma única um tipo de dispositivo (isto pode ser recuperado usando o gamepad_get_guid() função)
  • description - uma descrição Ascii do dispositivo (isto pode ser recuperado usando o gamepad_get_description() função)
  • platform-name - será ou "platform:android", "platform:windows", "linux" ou "mac" (note que isto será anexado automaticamente ao string quando você chamar esta função, e assim você não precisará fornecê-lo ao construir seu próprio mapeamento strings)
  • bindings - um conjunto de entradas separadas por um ',' para vincular a entrada real a constantes específicas GML

A lista de entradas separadas por vírgulas segue um formato específico para cada entrada:

<abstract-gp-name>:<value-binding>

Aqui <value-binding> (ou seja: a entrada direta do gamepad) pode ser qualquer uma das seguintes:

  • a<N> - Isto representa um eixo do gamepad, onde o <N> é um dos eixos de 0 a gamepad_axis_count() -1
  • b<N> - Isto representa um botão do gamepad, onde o <N> é um dos botões de 0 a gamepad_button_count() -1
  • h<N>.<mask> - Isto representa um chapéu do gamepad, onde o <N> representa um chapéu de 0 a gamepad_hat_count() -1, e o <mask> será ou 1, 2, 4, ou 8.

Cada valor de entrada deve ser vinculado a um nome SDL, que é expresso como <>abstract-gp-name>, que por sua vez está vinculado a uma constante GML. A tabela abaixo mostra a equivalência entre o nome SDL e a constante GameMaker:

Quando você tiver construído seu mapeamento string, ele deve se parecer com algo assim:

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,rightshoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

Sintaxe:

gamepad_test_mapping(index, mapping_string);

Argumento Tipo Descrição
index Real Which gamepad index "slot" to set.
mapping_string Cordão The map string to use (see the description for more information).

Devoluções:

N/A

Exemplo:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for (i = 0; i < len; i += 2)
{
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping);

O código acima será loop através de várias instâncias e utilizará os valores de diferentes variáveis que elas contêm para criar um mapeamento string, que é então definido para uso no gamepad no índice do slot dado.
