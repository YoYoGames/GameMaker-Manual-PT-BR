network_send_broadcast

Com esta função você pode transmitir os dados de um buffer localmente para uma gama de endereços IP (para mais informações em buffers veja Referência - Buffers). A faixa é limitada à do dispositivo executando o server, de modo que se o dispositivo tiver um IP de 92.168.11.130, então os dados serão transmitidos na faixa 92.168.11.*. A função retornará o número de bytes de dados enviados, ou um número inferior a 0 se o envio falhar.

Esta função só funcionará quando usada com UDP - seu server precisa ser TCP e seu cliente precisa ter um soquete cliente UDP criado com network_create_socket_ext() a fim de receber quaisquer transmissões enviadas do site server.

Esta função não funcionará quando utilizada em um projeto em execução no alvo HTML5.

Sintaxe:

network_send_broadcast(socket, port, bufferid, size);

Argumento Tipo Descrição
socket ID da tomada de rede The id of the socket to use.
port Real The port that the server will use.
bufferid Identificação do tampão The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Devoluções:

Real

Exemplo:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

O código acima escreve o nome string do atual server (armazenado em "global.ServerName"), depois escreve-o em um binário buffer com o id "broadcast_buffer". Estes dados são então transmitidos localmente para uma faixa de IPs (o IP do dispositivo está atualmente implícito como a faixa base de transmissão) para a porta 6511.
