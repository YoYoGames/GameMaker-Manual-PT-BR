ponto_de_colisão

Collision_point verifica o ponto especificado pelos argumentos x1,y1 para uma colisão com qualquer instância do object especificado pelo argumento "obj". esta verificação pode ser precisa ou não, mas para que colisões precisas sejam ativadas, o object ou instância que você está verificando também deve ter colisões precisas ativadas para seu sprite. Caso contrário, a verificação padrão é baseada em caixas delimitadoras. A imagem a seguir ilustra como isto funciona:

Lembre-se, para que colisões precisas sejam consideradas tanto a object sprite como a função de colisão devem ter uma marcação precisa como em. Deve-se observar também que o valor de retorno da função pode ser o id de qualquer uma das instâncias consideradas em colisão, portanto, se três instâncias se sobrepuserem naquele ponto, qualquer um de seus ids pode ser o valor de retorno da função.

Note que ao invés de um índice object você pode fornecer uma instância id para verificar uma instância específica, ou as palavras-chave da instância all, ou other (dependendo do evento e do escopo do código atual).

Sintaxe:

collision_point(x, y, obj, prec, notme);

Argumento Tipo Descrição
x Real The x coordinate of the point to check.
y Real The y coordinate of the point to check.
obj Objeto Ativo ou ID da instância The object to check for instance collisions.
prec Booleano Whether the check is based on precise collisions (trueque é mais lento) ou sua caixa de delimitação em geral (false, mais rápido).
notme Booleano Whether the calling instance, if relevant, should be excluded (true) ou não (false).

Devoluções:

Instance ID or noone

Exemplo:

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10;
}

Aqui estamos verificando o ponto na posição do object que tem o código para o object "obj_Cursor". Se houver um, então adicionamos 10 na variável de pontuação.
