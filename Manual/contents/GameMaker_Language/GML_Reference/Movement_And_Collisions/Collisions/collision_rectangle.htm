colisão_rectângulo

Collision_rectangle usa os primeiros quatro argumentos (x1,y1,x2,y2) para definir uma área dentro do atual room e depois verifica se algum object que é definido pelo argumento "obj" está em colisão com essa área. Esta colisão pode ser verificada como precisa ou não, e você também pode optar por verificar, por exemplo, se está ou não executando o código em si. Considere esta imagem:

Aqui, o exemplo no meio é usar um retângulo de colisão para verificar a bola objects. Agora, os azuis não têm uma caixa de delimitação precisa e, como você pode ver, mesmo que o sprite não esteja realmente tocando o retângulo, a colisão ainda acontecerá (mesmo que você defina a opção precisa na função como verdadeira) como a caixa de delimitação daquele sprite sobrelaça o retângulo de colisão_correção. Por outro lado, as bolas verdes só serão consideradas em colisão se o próprio sprite sobrelaçar o retângulo. Lembre-se, para que colisões precisas sejam consideradas tanto o object sprite quanto a função de colisão devem ter a marcação precisa como em. Também deve ser observado que o valor de retorno da função pode ser o id de qualquer uma das instâncias consideradas como em colisão.

Note que ao invés de um índice object você pode fornecer uma instância id para verificar uma instância específica, ou as palavras-chave da instância all, ou other (dependendo do evento e do escopo do código atual).

Sintaxe:

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

Argumento Tipo Descrição
x1 Real The x coordinate of the left side of the rectangle to check.
y1 Real The y coordinate of the top side of the rectangle to check.
x2 Real The x coordinate of the right side of the rectangle to check.
y2 Real The y coordinate of the bottom side of the rectangle to check.
obj Objeto Ativo ou ID da instância The object to check for instance collisions.
prec Booleano Whether the check is based on precise collisions (trueque é mais lento) ou sua caixa de delimitação em geral (false, mais rápido).
notme Booleano Whether the calling instance, if relevant, should be excluded (true) ou não (false).

Devoluções:

Instance ID or noone

Exemplo:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if (inst != noone)
{
    instance_destroy(inst);
}

Este código curto usa a verificação do ângulo de colisão_reta angular uma área no room de 50x, 50y (parte superior esquerda do retângulo) a 200x, 200y (parte inferior direita do retângulo) para uma instância de um object chamado "obj_ball". Ele armazena o valor de retorno em uma variável temporária que é então verificada para ver se esse valor é um id de instância, ou a palavra-chave noone. Se não for ninguém, então usa a identificação da instância armazenada para destruir o object.
