HTTP

Esta seção lista todas as diferentes funções assíncronas HTTP disponíveis no GameMaker. Estas funções gerarão um evento HTTP assíncrono em todas as instâncias que o possuem:

  • http_request
  • http_get
  • http_get_file
  • http_post_string

Favor observar que as funções http_* acima podem não funcionar como esperado devido a questões de segurança entre domínios. Isto significa que as solicitações ao seu server ou tentativas de carregar recursos de domínios cruzados são bloqueadas e podem parecer retornar resultados em branco ou 404 erros. Uma das maneiras que você pode contornar isto é ter algum server lado PHP que permite que certos domínios acessem seu server (esta também é uma maneira de proteger seus recursos e bloquear servers que não estão incluídos na lista de permissão de PHP).

A seguir, um exemplo de código PHP que você pode usar para isto:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
    header("Access-Control-Allow-Origin: *");
}

Para pedidos de carga de imagens onde a determinação ou a configuração de seu tipo de origem cruzada é importante, existem as seguintes funções:

  • http_get_request_crossorigin
  • http_set_request_crossorigin

Na maioria dos casos, estas funções não devem precisar ser usadas, mas se o jogo for armazenado em um site seguro server - onde certos assets podem exigir autenticação básica para serem acessados e estão gerando erros de segurança ao carregar - pode ser necessário definir o tipo de origem cruzada para pedidos de imagens como "credenciais de uso" (ao contrário da configuração padrão "anônima").
