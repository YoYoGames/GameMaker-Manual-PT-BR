show_message_async

Esta função abre uma janela e exibe ao usuário a mensagem que você define na função. Esta é uma função assíncrona, e como tal o GameMaker não bloqueia o dispositivo em que está sendo executado enquanto espera por uma resposta, mas sim continua a executar eventos como de costume. Uma vez que o usuário tenha pressionado o botão "Ok", uma função assíncrona Diálogo é acionado que, somente durante a duração desse evento, terá um ds_map armazenado na variável async_load.

Este mapa conterá as duas teclas, "id" e "status", sendo "id" o valor que foi retornado pela função quando chamada, e o "status" sendo ou true para o botão "Ok" sendo pressionado, ou false se a mensagem foi cancelada (quando disponível, pois nem todas as plataformas alvo permitem o cancelamento de diálogos).

NOTA Esta função é apenas para depuração e testee não deve ser usada em jogos lançados. Para isso, você deve criar uma interface de usuário personalizada para receber informações dos jogadores, para que você tenha controle total sobre a aparência e o comportamento dos diálogos.

Sintaxe:

show_message_async(string);

Argumento Tipo Descrição
string Cordão The message to show to the user.

Devoluções:

Async Request ID

Exemplo ampliado:

O evento de pressão do mouse esquerdo (por exemplo) do object que está mostrando a mensagem teria o seguinte código:

msg = show_message_async("You will now be taken to the store");

O acima exposto mostrará uma mensagem com o dado string. A identificação da mensagem é armazenada na variável "msg" e será usada no evento de Diálogos Assíncronos, como mostrado abaixo:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        url_open("https://play.google.com/store");
    }
}

O código acima verifica a chave "id" do retorno DS Map em relação ao valor armazenado na variável "msg". Se forem os mesmos, verifica se o botão "Ok" foi pressionado (em vez da janela ser fechada/cancelada) e se ele retorna true abre uma url.
