Constantes de sombreamento

Além das funções e constantes shader definidas nas Páginas de Referência OpenGL ES Shading Language (GLSL ES), há também um número de constantes shader disponíveis que são exclusivas do GameMaker.

As constantes de matriz de exibição a seguir podem ser usadas como índices de matriz ao usar a constante de matriz shader gm_Matrices :

Constante Descrição
MATRIX_VIEW Esta constante de índice de matriz mantém o índice com a matriz de visão atual. O índice retornado seria usado como um valor de matriz ao chamar a constante gm_Matrices dentro do código shader.
MATRIX_PROJECTION Esta constante de índice de matriz mantém o índice na matriz de projeção atual. O índice retornado seria usado como um valor de matriz ao chamar a constante gm_Matrices dentro do código shader.
MATRIX_WORLD Esta constante de índice de matriz mantém o índice na matriz mundial atual. Isto pode ser usado para coisas como iluminação, se você tiver informações de luz no espaço mundial. O índice retornado seria usado como um valor de matriz quando se chama a constante gm_Matrices dentro do código shader.
MATRIX_WORLD_VIEW Esta constante de índice de matriz mantém o índice com o resultado do mundo e as matrizes de visão multiplicadas juntas. Isto é freqüentemente usado para coisas como nevoeiro. O índice retornado seria usado como um valor de array ao chamar a constante gm_Matrices dentro do código shader.
MATRIX_WORLD_VIEW_PROJECTION Esta constante de índice de matriz mantém o índice com o resultado do mundo, matrizes de visão e projeção multiplicadas juntas. Esta é a matriz de transformação normal utilizada para as posições vertex. O índice retornado seria usado como um valor de matriz ao chamar a constante gm_Matrices dentro do código shader.
MATRIX_MAX Este não é um índice de matriz, mas retorna o tamanho da matriz no site vertex shader.
MAX_VS_LIGHTS Este não é um índice de matriz, mas retorna o número das luzes no vertex shader.

A seguinte constante também está disponível para iluminação:

Constante Descrição
MAX_VS_LIGHTS O número máximo de luzes pontuais e direcionais disponíveis no sombreador

Os seguintes uniformes de matriz pré-definidos e constantes podem ser usados em seu shader para acessar valores específicos do GameMaker:

Constante Descrição
gm_Matrices[matrix] Esta constante de matriz retorna uma matriz de transformação e é um dos uniformes pré-definidos disponíveis que o GameMaker cria para você usar dentro do editor de código shader. O índice de matriz é escolhido a partir de uma das constantes listadas acima.
gm_BaseTexture Esta é uma constante de amostragem 2D que retorna o texture do atual object, conforme definido pelo GameMaker. Assim, seria a atual sprite, superfície ou texture que normalmente seria usada ao desenhar sem que o shader fosse chamado.
gm_LightingEnabled Isto pode ser usado para obter ou definir a iluminação GameMaker ao usar o 3D.
gm_FogStart Isto pode ser usado para obter a distância onde os polígonos começam a ser misturados com a cor da neblina.
gm_RcpFogRange Isto pode ser usado para obter a distância a que a neblina é máxima e nada mais pode ser visto.
gm_PS_FogEnabled Isto retornará true ou false se a GPU tiver nevoeiro de pixel habilitado ou não.
gm_FogColour Isto pode ser usado para que a cor da neblina seja usada pelo GameMaker.
gm_VS_FogEnabled Isto retornará true ou false se a GPU tiver vertex nevoeiro ativado ou não.
gm_AlphaTestEnabled Isto pode ser usado para obter o teste alfa no site shader. Veja gpu_set_alphatestenable() para mais informações sobre os testes alfa.
gm_AlphaRefValue Isto pode ser usado para obter o valor de referência atual do teste alfa. Ver gpu_set_alphatestref() para obter mais informações sobre a referência do teste alfa.
