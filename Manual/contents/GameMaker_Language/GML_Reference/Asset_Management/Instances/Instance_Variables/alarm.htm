alarme

Esta matriz de 1 dimensão é usada para obter o valor atual para quaisquer alarmes que a instância possa ter, ou pode ser usada para definir esses alarmes. Há doze alarmes embutidos em cada instância de um object, e cada um tem seu próprio event que será executado quando esta variável chegar a 0.

Os alarmes são contados no início de cada etapa, após o evento da etapa inicial, mas antes do evento da etapa principal, e antes que qualquer coisa seja entregue para o quadro. Isto significa que se você definir um alarme para 1 no evento Begin Step, seu evento será executado nesse mesmo quadro (pois o alarme será contado até 0 após o término do Begin Step), mas se você definir um alarme para 1 no evento Step, ele será executado no próximo quadro.

Deve-se observar que o alarme não está terminado quando chega a 0 (embora o evento tenha sido acionado), pois o próximo passo será descer para -1, portanto, se você precisar parar um alarme por qualquer motivo, você deve definir seu valor de matriz para -1 e não 0. Os tempos de alarme são calculados em etapas de jogo, com um valor de 30 sendo 30 passos, 60 sendo 60 passos, etc...

OBSERVAÇÃO Um alarme sem ações ou código nele não será executado. Entretanto, mesmo com apenas um comentário e sem código, o alarme será contado para baixo.

Você pode definir a matriz de alarmes diretamente (e recuperar o valor atual do alarme diretamente usando a matriz também), mas sob algumas circunstâncias este não é o método mais apropriado, então você também pode usar a função alarm_set() para definir um alarme, e a função alarm_get() para obter o valor de um alarme.

Sintaxe:

alarm[val];

Devoluções:

(-1 if the alarm isn't running)

Exemplo:

if canshoot
{
    if (keyboard_check_pressed(vk_space))
    {
        canshoot = false;
        alarm[0] = room_speed;
        instance_create_layer(x, y, "Bullets", obj_Bullet);
    }
}

O código acima verifica uma variável e então, se for verdadeira, verifica se há uma tecla pressionada e se for verdadeira, finalmente define a variável como falsa, define um alarme (onde ela será definida como verdadeira novamente) e cria uma instância.
