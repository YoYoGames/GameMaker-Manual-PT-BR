A estrutura da via

A faixa struct pode ser usada tanto parafaixas deativos denível superior quanto parafaixas de parâmetros de sub-faixa , e o comportamento da faixa é definido por duas coisas, seu nome e seu tipo. As estruturas da pista são criadas usando a função sequence_track_new() e pode ser recuperado em sequence assets usando a propriedade activeTracks da Sequence Instance Struct ou a propriedade tracks da Sequence Object Struct.

As propriedades disponíveis na pista struct são:

Estruturas de trilhos seqüenciais
Variável Tipo Descrição
type Constante do tipo de pista sequencial Isto contém um Constante do tipo de pista sequencial que descreve o tipo de trilha que isto é. Esta pode ser qualquer uma das constantes dadas sob a seção "Tipo" abaixo.
name Cordão Ao criar uma faixa "top-level" asset, o nome que você dá aqui pode ser qualquer string que você precisar para identificar a faixa. Entretanto, para as faixas de parâmetros, você precisa especificar strings específico para dizer ao GameMaker que tipo de faixa de parâmetros você está criando. Isto é expandido na seção "Nome" abaixo.
tracks Array de Estruturas de trilhos seqüenciaiss Esta propriedade permite o acesso à lista de faixas que são crianças desta faixa. Ao obter este imóvel, um conjunto de Estruturas de trilhos seqüenciaiss é devolvido, e ao definir esta propriedade uma série de Estruturas de trilhos seqüenciaiss deve ser especificado.
visible Booleano Isto indica se esta faixa é visível (o valor é true) ou não (o valor é false). Você pode obter ou definir este valor e se uma faixa não for visível, nenhuma de suas faixas infantis será desenhada também.
keyframes Array de Sequência Keyframe Estruturass Esta propriedade permite o acesso à lista de estruturas do quadro-chave para a pista. Ao obter esta propriedade, um array de keyframe structs é retornado, e ao definir esta propriedade, um array de keyframe structs deve ser especificado.
enabled Boolean This indicates whether this track is enabled. If the track is an asset track it isn't drawn when disabled, if it's a parameter track the track's values aren't applied.

Nome

A propriedade name pode ser qualquer uma das seguintes cordas:

  • "posição" - quando colocado em uma pista do tipo seqtracktype_real, isto indica que o conteúdo desta pista deve ser usado como dados de posicionamento.
  • "escala" - quando colocado em uma pista do tipo seqtracktype_real, isto indica que o conteúdo desta pista deve ser usado como dados de escala.
  • "rotação" - quando colocado em uma pista do tipo seqtracktype_real isto indica que o conteúdo desta pista deve ser usado como dados de rotação.
  • "blend_multiply" ou "image_blend" - quando colocado em uma pista do tipo seqtracktype_colour isto indica que o conteúdo desta pista deve ser usado como dados de multiplicação de cores. Os dados keyframe para tal faixa devem ser uma matriz de quatro valores com o formato [A, R, G, B]. Note que os valores para cada componente são expressos entre 0 e 1, onde 0 corresponde ao valor HEX #00 e 1 corresponde ao valor HEX #FF (0 - 255 como mostrado no seletor de cores para faixas de mistura de imagens no Editor Sequences ).
  • "image_speed" - quando colocado em uma pista do tipo seqtracktype_real isto indica que o conteúdo desta pista deve ser usado como o valor da velocidade da imagem.
  • "image_index" - quando colocado em uma faixa que é do tipo seqtracktype_real isto indica que o conteúdo desta faixa deve ser usado como o valor do índice de imagem.
  • "image_angle" - quando colocado em uma faixa que é do tipo seqtracktype_real isto indica que o conteúdo desta faixa deve ser usado como o valor do ângulo da imagem.


    Nomes de parâmetros de trilha de texto (usados somente em Trilhas de texto):
  • "frameSize" - quando colocado em uma faixa seqtracktype_real, isto indica que o conteúdo desta faixa deve ser usado como o tamanho do quadro para o texto.Cada keyframe para esta faixa deve ter dois canais, 0 e 1, onde 0 armazena o tamanho X e 1 armazena o tamanho Y do quadro.
  • "characterSpacing" - também usado com faixas seqtracktype_real, usado como espaçamento de caracteres para o texto. Cada keyframe para esta faixa deve ter um único canal contendo o valor do espaçamento.
  • "lineSpacing" - também usado com faixas seqtracktype_real, usado como espaçamento de linha para o texto. Cada keyframe para esta faixa deve ter um único canal contendo o valor do espaçamento.
  • "parágrafoSpacing" - também usado com faixas seqtracktype_real, usado como espaçamento de parágrafo para o texto. Cada keyframe para esta faixa deve ter um único canal contendo o valor do espaçamento.
