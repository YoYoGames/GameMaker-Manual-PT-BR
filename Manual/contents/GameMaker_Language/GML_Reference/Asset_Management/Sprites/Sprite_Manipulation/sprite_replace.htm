sprite_replace

Esta função funciona quase exatamente da mesma maneira que sprite_add()somente em vez de devolver o índice do sprite que você está importando, ele sobrescreve um índice previamente criado sprite. Ao usar esta função você deve usar um índice sprite que foi criado e armazenado em uma variável usando outras funções como sprite_add() ou sprite_create_from_surface()ou mesmo sprite_duplicate()em vez de uma árvore de recursos sprite asset. Você pode substituir um sprite do jogo assets usando esta função, mas isso significa que você perderá o ID de referência para o sprite que você está substituindo, independentemente de ligar para game_restart() ou não, e por isso não é recomendado. Independentemente do sprite ser substituído, esta função criará uma nova página texture para o sprite e, portanto, deve-se ter cuidado ao utilizá-lo, pois pode afetar negativamente o desempenho ao aumentar o número de trocas necessárias de texture para renderização.

O arquivo de imagem a ser carregado deve estar sempre no formato *.png e todas as imagens a serem transformadas em animadas sprites devem ter um formato "strip" (veja a imagem abaixo). Elas serão divididas no número de sub-imagens especificadas de acordo com a regra sprite width = strip width / sub images.

Como você pode ver na imagem acima, o sprite foi colocado sobre um fundo roxo escuro, e isto pode ser removido colocando o argumento "removeback" em true. Isto funciona verificando o pixel inferior esquerdo do sprite para a cor lá e depois usa isso como a cor a ser removida. Por exemplo, na imagem acima, se tivéssemos a cor do píxel inferior esquerdo como verde, todas as partes verdes do sprite teriam sido removidas e o resto do fundo roxo teria sido ignorado.

Se você escolher a opção "remover", você também pode querer que o GameMaker suavize as bordas do sprite, definindo o argumento "suavizar" para true. Tudo o que isso faz é criar uma borda semi-transparente ao redor das bordas do sprite após ter tido seu fundo removido.

Finalmente, você também pode especificar a origem x e y para o sprite. Este é o ponto onde o sprite é "fixo" na instância que o utiliza, e é sempre calculado como relativo ao canto superior esquerdo 0,0 de uma subimagem sprite. Assim, por exemplo, um sprite que é 32 x 32 pixels com estes valores definidos para (16,16) terá sua origem no centro.

Por padrão, todos os novos sprites têm suas caixas delimitadoras calculadas automaticamente (a caixa exata dependerá do tamanho e da transparência do sprite), entretanto você pode querer personalizar isso, caso em que você também deve usar a função sprite_collision_mask().

NOTA: Dependendo da plataforma de destino escolhida, você está limitado quanto ao local de onde pode salvar e carregar arquivos. Consulte Manuseio de arquivos para mais informações.

NOTA: Você deve estar ciente de que se você estiver usando esta função em seu jogo de destino HTML5 para carregar recursos de um server externo, então, devido à proteção XSS em navegadores, as tentativas de carregar recursos de vários domínios podem ser bloqueadas e podem parecer retornar resultados em branco.

Sintaxe:

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

Argumento Tipo Descrição
ind Ativo Sprite The index of the sprite to permanently replace.
fname Cordão The filename of the image to make the new sprite.
imgnumb Real The number of frames the sprite will be cut up into horizontally. Use 1 for one single image or *.gif.
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges.
xorig Real The x coordinate of the origin, relative to the sprite's top left corner.
yorig Real The y coordinate of the origin, relative to the sprite's top left corner.

Devoluções

N/A

Exemplo:

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

O código acima substituirá a imagem asset indexada em "spr_banner" por outra carregada a partir de uma fonte externa.
