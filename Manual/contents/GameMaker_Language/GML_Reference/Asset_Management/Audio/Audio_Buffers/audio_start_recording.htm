audio_start_recording

Esta função começará a gravar áudio a partir da fonte indexada do gravador. Você pode obter o número de fontes do gravador usando a função audio_get_recorder_count()e assim que você começar a gravar o áudio será armazenado em um buffer temporário e começará a disparar um Evento Assíncrono de Gravação de Áudio. Este evento é acionado a cada etapa da gravação e criará o mapa DS especial na variável async_load com os seguintes pares de chaves/valores:

  • "buffer_id" - a identificação do provisório buffer que você pode usar para recuperar os dados de áudio
  • "channel_index" - o índice do canal de gravação (como retornado pela função) de onde vieram estes dados
  • "data_len" - o comprimento dos dados (em bytes) que você recebeu

Note que após o evento assíncrono ter sido acionado, o áudio no buffer temporário será apagado, portanto você deve copiar seus dados em um buffer personalizado que você criou anteriormente.

NOTA A maioria das plataformas suporta a gravação de áudio de alguma forma, mas isso não significa que todos os dispositivos o permitam, mesmo que a plataforma o permita, então você deve sempre verificar se o audio_get_recorder_count() retorna um valor maior que 0 para verificar se os dispositivos de gravação estão disponíveis antes de utilizar o restante das funções de gravação.

Sintaxe:

audio_start_recording(recorder_index);

Argumento Tipo Descrição
recorder_index Real The index of the recorder source to use.

Devoluções:

Buffer ID

Exemplo:

audio_record = audio_start_recording(0);

O código acima começa a gravar a partir da fonte do gravador 0, armazenando o índice de canal da gravação na variável "audio_record" para uso no evento assíncrono de gravação de áudio.
