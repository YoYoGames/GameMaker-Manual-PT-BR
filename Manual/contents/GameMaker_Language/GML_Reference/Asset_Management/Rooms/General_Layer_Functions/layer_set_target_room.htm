layer_set_target_room

Quando você chama esta função, você está dizendo ao GameMaker que todas as outras funções de camada devem ser aplicadas ao espaço em questão. Desta forma, você pode mudar ou gerar camadas e conteúdos de camadas em um room que não é o atual room. Quando você terminar de adicionar camadas ou elementos de camadas em um room, chame a função layer_reset_target_room() para redefinir o alvo room (ou chamar esta função novamente com um argumento room de -1). Note que esta função só pode ser usada em salas diferentes da sala atual, e é projetada para que você possa adicionar/remover camadas e elementos de camadas para rooms além do room que está atualmente em execução.

ADVERTÊNCIA! Ao visar outro room, você pode usar todas as funções de camadas regulares , exceto que você não pode criar instâncias usando instance_create_layer() ou instance_create_depth()nem a camada funcionará layer_add_instance() estar disponível.

Sintaxe:

layer_set_target_room(room)

ArgumentoTipo Descrição
room The room to target for all further layer functions

Devoluções:

Exemplo:

layer_set_target_room(rm_Game);
var l = layer_get_id("SpriteAssets");
repeat(50)
{
    var _x = irandom(1000);
    var _y = irandom(1000);
    layer_sprite_create(l, _x, _y, spr_Trees);
}
layer_reset_target_room();

O código acima estabelece a meta room para o room "rm_Game" e então obtém a identificação da camada chamada "SpriteAssets" naquele room. Este ID de camada é então usado para adicionar 50 aleatórios sprite assets à camada, antes que o alvo da camada seja redefinido para o atual room.
