Funções em tempo de execução

A definição geral de uma função é algo como isto:

Uma função tem uma entrada e uma saída, e a saída está relacionada com a entrada.

No GameMaker isso se traduz em duas coisas diferentes, mas ambas funcionam da mesma maneira:

  • Funções de tempo de execução - Uma função de tempo de execução é aquela que é fornecida como parte da linguagem GameMaker (GML). Elas são integradas ao idioma e podem ser usadas para fazer as coisas acontecerem em seus jogos e há um grande número de funções GML runtime disponíveis para você, todas explicadas na seção Referência do Idioma do manual.
  • Funções e métodos do roteiro - Roteiro funções e Mvariáveis de ethod são funções que você mesmo criou usando os blocos básicos de construção da linguagem GameMaker (GML), e estas podem incluir runtime funções como parte de seu código.

Em ambos os casos acima, uma função tem a forma de um nome de função, seguido pelos argumentos de entrada entre colchetes () e separado por vírgulas (se a função não tiver argumentos de entrada, são usados apenas colchetes). Esta página concentra-se nas funções de tempo de execução (ou seja: as que estão embutidas em GameMaker para formar o GameMaker Language), mas as regras gerais mostradas abaixo seguem para todos os tipos de funções.

Aqui está um esboço da estrutura de uma função:

<function>(<arg0>, <arg1> ,... <argN>);

Algumas funções podem retornar valores e podem ser usadas emexpressões, enquanto outras simplesmente executam comandos, como ilustrado nos dois exemplos de funções runtime a seguir:

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

NOTA Em qualquer lugar onde o manual tenha "N/A" como valor de retorno para uma função, significa que a função não tem a intenção de retornar nenhum valor como parte de sua operação. Se você tentar verificar um valor de retorno destas funções, então você receberá o valor undefined.

s vezes o valor de retorno de uma função pode ser um valor que você deseja usar em uma atribuição, mas você deve observar que é impossível usar qualquer função diretamente como o lado esquerdo de uma atribuição. Por exemplo, o seguinte código para definir a velocidade em uma instância lhe daria um erro:

instance_nearest(x, y, obj).speed = 0;

O valor de retorno para a expressão nesse exemplo de código é um número inteiro (o valor de identificação único para a instância mais próxima) e, portanto, deve ser incluído entre parênteses para ser usado desta forma e endereçar corretamente a instância requerida (ver Variáveis de endereçamento em outras instâncias para mais informações). O código acima seria escrito corretamente como:

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

A seção Referência Linguística do manual lista todas as funções runtime disponíveis e dá exemplos de como elas podem ser usadas, bem como outras informações importantes sobre o que elas podem retornar ou quaisquer eventos que elas possam desencadear, etc.. Para mais informações sobre funções e métodos definidos pelo usuário script, consulte a seção sobre Funções do Script e a seção sobre Variáveis de Métodos.
