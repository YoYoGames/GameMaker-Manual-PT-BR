Todos os jogos que você faz no GameMaker precisam de pelo menos um room para rodar (mas podem ter muitos, muitos mais) , e um room é apenas um espaço onde você coloca instâncias do objects que compõem seu jogo, assim como tiles e quaisquer outros recursos gráficos. Quando você criar pela primeira vez um room no Navegador de Ativos, você será presenteado com um novo Editor de sala workspace com o qual editar suas propriedades, algo como isto:

Por padrão, as ferramentas room serão colocadas à esquerda, com três seções principais:

  • Editor de camadas: Tudo em um room é colocado em camadas, e há diferentes tipos de camadas que você pode escolher (mais sobre isso mais adiante). As camadas são ordenadas por profundidade e esta profundidade é o que define a ordem em que o conteúdo da camada será apresentado na tela quando o jogo correr. A ordem de profundidade é da mais alta para a mais baixa, de modo que quanto mais baixa a profundidade, mais perto da "câmera" e quanto mais alta a profundidade, mais longe. Por exemplo, uma camada com uma A profundidade de -300 renderá sobre uma camada com uma profundidade de -100, e uma camada com uma profundidade de 1000 renderá sob tudo com uma profundidade menor do que esta. Note que por padrão, o GameMaker ordenará as profundidades de camada para você de acordo com a posição no Layer Editor, e você pode arrastar as camadas usando o mouse para reordená-las e o GameMaker reordenará as profundidades para combinar.
  • Propriedades de camadas: Cada camada que você adicionar ao room terá suas próprias propriedades, e essas propriedades mudarão dependendo do tipo de camada. Esta janela permite mudar essas propriedades e editar como a camada será prestados.
  • Propriedades da sala: Rooms também tem uma série de propriedades, e estas podem ser definidas aqui. Coisas como o tamanho room, as portas de visualização da câmera que estão ativas e algumas outras coisas podem ser configuradas aqui.

Mencionamos que existem diferentes tipos de camadas, então vamos rever o que elas são e como podem ser usadas na construção de seus jogos:

Camada de fundo A camada de fundo é uma camada que pode ser preenchida com uma única cor ou uma única imagem. Ela pode ser movida e posicionada dentro do room, e você pode ter várias camadas de fundo. Geralmente isto é usado, como o nome sugere, para gerar um fundo constante para todas as outras camadas no room. Por padrão, um novo room sempre conterá uma camada de fundo, mas você pode removê-la se não precisar dela.
Camada de instância A camada de instância é onde você coloca todas as instâncias do objects que você precisa para o jogo. Para adicionar uma instância a uma camada, basta clicar no endereço object no asset navegador e depois arraste-o para o editor room workspace . Quando você soltar o botão do mouse, uma instância desse object terá sido adicionada ao room. Por padrão, um novo room sempre conterá uma camada de instância, mas você pode removê-la. se você não precisar dele.
Camada do mapa de azulejos Após ter criado um conjunto de azulejos, você precisa adicionar o tiles ao seu room, o que é feito através da criação de uma camada de mapa de azulejos. Uma camada do mapa tile é uma camada que permite adicionar tiles a partir de qualquer um dos tile set assets que você criou e será configurada automaticamente para usar uma grade do tamanho das células tile set. Você só pode adicionar um único tile a cada célula da grade, portanto, se você precisar de múltiplos tiles para ocupar o mesmo espaço, então você deve estar usando múltiplos tile camadas do mapa.
Camada do caminho A camada path é a única que não é realmente renderizada quando você executa seu jogo. Esta camada é mais uma camada de "conveniência" para ajudá-lo a criar ou editar ativos do Caminho , uma vez que permite adicionar ou editar paths dentro do próprio espaço room.
Camada de ativos Às vezes você quer um bom efeito gráfico, mas não quer a sobrecarga de usar uma instância ou não precisa que ela faça nada além de desenhar-se a si mesma. Você poderia usar um mapa tile, mas dado que os mapas tile são restritos ao uso de uma grade e apenas uma única imagem por célula de grade, elas podem ser um pouco restritivas. Isto é quando você usaria a camada asset. A camada asset simplesmente pega um recurso sprite e o desenha usando os parâmetros que você define quando o adiciona no room. Sprites e sequences pode ser adicionado a esta camada da mesma forma que as instâncias de objects são adicionadas à camada de instância, ou seja: você clica no sprite no navegador asset e depois arraste-o para o editor room workspace .

Não entraremos em muito mais detalhes sobre o editor room aqui (para isso temos esta seção do manual), mas explicaremos brevemente como adicionar instâncias a uma camada de instância, pois você precisará saiba que para a próxima seção do Guia de Início Rápido.

Para adicionar uma instância ao editor room, você deve primeiro selecionar uma camada de instância do editor de camada à esquerda (ou criar uma se não existir nenhuma). Em seguida, você pode ir ao navegador Asset e simplesmente clicar em e arrastar um object que você queira de lá e soltá-lo no room na posição desejada, o que adicionará uma única instância desse object ao seu jogo:

Se você quiser colocar múltiplas instâncias no room, então você pode selecionar um object da árvore de recursos e então pressionar e segurar para visualizar o recurso, e se você clicar adicionalmente no botão esquerdo do mouse você pode então "pintá-la" para a camada room.

Observe que assets recebem uma posição quando são adicionados ao editor room com base em suas posições dos eixos X e Y em relação à origem room. A origem do room é a posição (0, 0), que no GameMaker é o canto superior arrendado, e para a direita é +X e para baixo é +Y, por exemplo:

No exemplo acima, a instância é colocada no meio do room na posição (X:512, Y:384). Também é possível ir para fora do room e têm posições que são maiores que o comprimento do eixo room, ou mesmo menores que a origem room - neste caso, as coordenadas X/Y serão valores negativos.
