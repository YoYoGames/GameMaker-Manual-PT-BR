Objetos e Instâncias

Com o assets que você viu até agora, você pode adicionar algumas imagens e sons legais ao jogo asset browser, mas eles ainda não fazem nada no jogo em si! Para isso precisamos do assets mais importante de todos no GameMaker, o ativo Objeto . Objects é um asset especial que usamos para controlar aspectos de um jogo e para fazer coisas específicas. Na maioria das vezes eles têm um sprite associado a eles para que você possa vê-los no jogo rooms, mas às vezes eles são usados como objetos controladores "por trás dos bastidores" para fazer coisas como rastrear estatísticas, ou definir horários para cut-scenes, etc... Objects pode receber comportamentos e eles podem reagir a certos eventos assim como uns aos outros, e a maioria das coisas que você vê em um jogo são baseadas em objects e suas interações com o usuário e uns com os outros.

Agora, não podemos falar de Objects sem falar de Instâncias. Em um jogo feito com o GameMaker, os personagens, monstros, bolas, paredes, etc... que você vê no jogo são na verdade todos considerados como instâncias. Uma instância é essencialmente uma "cópia" de um object que é encontrada no navegador Asset e depois colocada em um room (pense em objects como modelos para instâncias). Portanto, quando falamos de algo que afeta ou altera uma instância, queremos dizer que uma cópia em particular de um object em um room está sendo afetada enquanto todas as outras não estão, mas quando falamos de afetar ou alterar um object, queremos dizer que qualquer coisa que fizermos a ele será refletida em todas as instâncias criadas a partir desse ponto também. Portanto, o object é o modelo para a instância, e a instância é o que colocamos em um room para fazer nosso jogo.

Para criar um novo object asset, é o mesmo que para qualquer outro asset e basta clicar com o botão direito do mouse na pasta Objects asset e selecionar Criar (ou clicar com o botão direito do mouse em qualquer lugar no Navegador de Ativos e selecionar Criar Objeto).

Esta é a janela do Object Editor e mostra as atuais propriedades object, e a primeira coisa que você deve fazer é nomear o object, novamente usando um prefixo como " obj_" ou algum outro identificador. Em geral, para que um object desenhe qualquer coisa e registre colisões com instâncias de outros objects, é necessário ter um sprite atribuído a ele, que é adicionado usando o botão . Isto abrirá o Asset Explorer onde você pode selecionar o sprite para usar (você também pode arrastar sprites do Navegador Asset e soltá-lo no Editor Object para atribuí-los):

Se você ainda não tem um sprite apropriado, você pode clicar no botão New Sprite para criar um novo sprite asset e editá-lo como desejar, assim como se você tivesse criado o asset através do padrão sprite criar opções, somente agora ele será automaticamente atribuído ao object que você tinha aberto quando você começou a criá-lo. Uma vez selecionado um sprite, você também pode clicar nos botões Editar Sprite ou Editar Imagem para editar as imagens do próprio sprite ou do sprite asset. Isto pode ser mais rápido do que encontrar primeiro o sprite no navegador Asset e depois indicar que você deseja editá-lo.

Uma vez que você tenha adicionado ou criado o sprite, você pode alterar algumas outras propriedades object como, por exemplo, se deve ser visível ou não - ou que tipo de máscara de colisão deve ter. Para jogos simples, você provavelmente não precisa mudar nada aqui e agora pode continuar a adicionar Eventos ao object.

Os eventos são momentos discretos no jogo loop onde as coisas são feitas para acontecer com base no que você programou para eles. O GameMaker trabalha com ciclos destes eventos - desde o momento em que um room é iniciado até o momento em que é terminado, há um jogo loop correndo onde a cada passo uma série de eventos é executada, e você pode escolher colocar o código GML ou GML Ações visuais em suas instâncias que respondem a estes eventos (um passo é um momento no tempo do jogo, governado pela configuração de velocidade do jogo - também chamado de "frame").

Para adicionar um evento, clique em no botão Adicionar Evento:

Acima você pode ver todas as diferentes Categorias de Eventos, algumas das quais contêm outras sub-categorias. Não passaremos por todos os eventos aqui, mas cobriremos brevemente os cinco mais importantes:

Criar Evento Este evento acontece quando uma instância do object é criada pela primeira vez, e é a primeira coisa que acontece dentro de uma instância que foi colocada no room através do editor room quando um room é inserido. Isto significa que este evento é o lugar ideal para inicializar variáveis, iniciar Timelines, configurar paths etc... e fazer qualquer outra coisa que geralmente só precisa ser feita uma vez ou somente quando uma instância aparece pela primeira vez no room.
Etapa Evento O GameMaker divide o tempo de jogo em etapas com a velocidade do jogo definindo quantas dessas etapas devem ser por segundo. Um passo, é basicamente o loop que corre constantemente com todos os eventos sendo verificados e acionados conforme necessário enquanto o jogo corre, de modo que, como você pode imaginar, o Evento de Passo é um evento que é verificado a cada passo do jogo enquanto a instância existe. Observe que a categoria Step Event tem três subcategorias para "afinar" o tempo dentro da instância, mas em geral você usará mais o Step Event principal.
Evento de colisão Obviamente, ao fazer um jogo, é muito importante que você saiba quando duas (ou mais) instâncias de um object colidiram, e para isso temos o evento da colisão. Este é um evento que você coloca em um object e depois especifica contra qual outro object você deve estar verificando se há colisões. Uma vez que você tenha adicionado o evento e escolhido o object para verificar, você pode adicionar o código GML ou GML Visual para resolver a colisão.
Evento de sorteio Existem vários eventos de sorteio diferentes dentro da categoria de eventos de sorteio, mas o que você mais usa é o que está no topo simplesmente chamado de "Sorteio". Este evento geral de sorteio será chamado, como o evento de etapa, cada quadro de jogo, mas funciona de duas maneiras diferentes. A primeira é quando você não adiciona este evento ao object... Quando não há um evento de sorteio especificado , então o GameMaker irá, por padrão, desenhar o sprite atribuído ao object, usando qualquer transformação que tenha sido aplicada em outro lugar no object (como para mudar seu tamanho ou mistura alfa), assim, mesmo quando nenhum evento de sorteio estiver listado na lista de eventos, ele ainda está sendo executado e ainda irá desenhar para a tela quando uma instância for colocada em um room. A segunda forma de funcionamento é quando você adiciona o evento em si e inclui qualquer GML ou GML Visual nele. Adicionar algo ao evento diz ao GameMaker " Eu quero controlar o que você desenha para as instâncias deste objeto" e ele sobrepõe completamente o sorteio padrão. Isto significa que você pode ter um object com um sprite atribuído, depois definir o evento do sorteio para o texto do sorteio e o sprite não será sorteado, pois você não disse ao GameMaker para sorteá-lo, embora o texto seja.
Evento Destroy Este evento é o último evento a ser executado quando uma instância é destruída. É freqüentemente negligenciado quando se adiciona comportamentos a objects, mas pode ser muito útil - por exemplo, para criar efeitos de explosão ou de partículas quando um inimigo é morto, ou para re-despertar uma nova instância do object em outra parte do room, ou mesmo para adicionar pontos a uma pontuação.
