Criando Sprites

Agora que você sabe o que é um programa, é hora de olhar para alguns dos diferentes ativos que podem ser usados no GameMaker para fazer seu jogo. Para começar, vamos olhar para os sprites, que são geralmente uma das primeiras coisas que você precisará ao fazer qualquer projeto. Como explicado na seção do The Asset Browser, um sprite é uma imagem que pode ser animada (embora não têm que ser) e depois atraídos para a tela. Em geral um sprite será associado a um object, mas você pode desenhar sprites por conta própria, seja a partir do código ou no editor room Asset Layer (falaremos sobre isto com mais detalhes mais tarde, na seção sobre quartos). Você também pode criar um sprite para ser usado como um conjunto de azulejos, mas exploraremos essa opção também em uma seção posterior.

Quando você cria um novo projeto a partir da página inicial, o navegador asset só conterá pastas vazias para os recursos que você possa precisar, então você precisa clicar com o botão direito do mouse na pasta sprite asset e então selecionar a opção Criar (se você não tiver uma pasta sprite asset você pode fazer uma a partir do menu do botão direito do mouse e então criar a sprite, ou você pode simplesmente clicar com o botão direito do mouse em qualquer lugar do ativo navegador e criá-lo). Isto criará um novo sprite asset e abrirá o editor sprite para você (se o editor sprite não abrir, basta clicar duas vezes em no novo asset):

Como você pode ver, a parte superior esquerda da janela tem um campo para o Nome do sprite. Todos sprites (e todos os outros assets) devem ter um nome dado a eles para que você (e GameMaker) possa identificá-los facilmente, embora você deva observar que este nome é realmente apenas um variável que possui um valor de identificação que "aponta" para o recurso, neste caso um sprite. É melhor dar a cada sprite um nome descritivo para que você possa identificar rapidamente se um determinado asset é um sprite ou um object ou algo mais, e muitas pessoas fazem isso prefixando ou sufixando o recurso com as letras " spr" - por exemplo, " spr_Ball". Observe que os nomes dos recursos estão limitados ao uso apenas de letras, dígitos e o sublinhado símbolo "_", e o nome de todos assets deve começar com uma letra e não com um número.

As outras características do editor sprite que discutiremos no final desta seção, mas primeiro precisamos explicar como desenhar um sprite. Isto é feito no Editor de imagem. A imagem editor é uma ferramenta muito poderosa para criar os gráficos em seu jogo e é aberta clicando no botão Edit Image .

Não vamos entrar em muita profundidade sobre as ferramentas disponíveis aqui - para isso temos a seção Editor de Imagem do manual - mas brevemente você tem:

  • Na parte superior esquerda, a vista da moldura - esta mostra a(s) moldura(s) de seu sprite e pode ser redimensionada usando os ícones da lupa.
  • No centro temos a tela de desenho - isto mostra a moldura atualmente selecionada pronta para desenho e você pode usar o botão central do mouse para dar a volta e a roda do mouse para aumentar e diminuir o zoom.
  • Na parte superior direita temos as escovas básicas - aqui você pode escolher a escova para desenho, e qualquer escova personalizada também será adicionada aqui.
  • No meio à direita temos o seletor de cores - aqui podemos escolher a cor a ser usada para o botão esquerdo e/ou direito do mouse, já que você pode desenhar na tela com um ou outro botão ao mesmo tempo.
  • Logo abaixo do colhedor de cores temos nossas ferramentas - estas são as diferentes ferramentas de desenho, incluindo a escova, o enchimento de inundação, o mascaramento e as ferramentas de forma. Observe que algumas das ferramentas são divididas diagonalmente e desenharão ou esboçadas ou preenchidas formas, dependendo do lado da linha em que você clicar para selecioná-las.
  • Na parte inferior direita temos a vista das camadas - como muitas outras ferramentas de desenho lá fora, você pode criar diferentes camadas para desenhar, permitindo que você experimente com posição e cor, etc... sem ter que se preocupar em destruir o que está por baixo.

Desenhe algo no editor de imagem e leve algum tempo para brincar com as opções, depois quando estiver pronto feche o editor de imagem workspace (que salvará a imagem) para levá-lo de volta para o principal workspace com o editor sprite nele. Vamos discutir algumas outras características do editor sprite que são importantes de se conhecer quando se está apenas começando...

A primeira coisa que você precisa saber é como definir a origem para o sprite. A origem é simplesmente o ponto que será usado para "ancorar" nosso sprite dentro do room, e neste caso queremos defini-la para o centro.

Portanto, clicamos em no menu inferior para definir a origem e selecionamos "centro do meio":

Você verá o marcador de origem azul na imagem de pré-visualização mover-se para o centro do sprite. Note que você pode colocar a origem em qualquer lugar que desejar, simplesmente clicando no imagem de pré-visualização, e você também pode inserir manualmente a posição de origem adicionando valores nos campos x/y na parte superior.

Outra coisa importante a saber é como definir a máscara de colisão para o sprite. A máscara de colisão é a área que o GameMaker verificará para ver se houve uma colisão com qualquer outra instância de um object no jogo, e, por padrão, isto é definido como retângulo (que também é o mais rápido a ser resolvido). Observe que você pode definir o modo de detecção para o caixa de delimitação para ser automático (o valor padrão) ou manual. Se você escolher manual você será capaz de definir os valores da caixa de delimitação esquerda, direita, superior e inferior.

Qualquer tipo de máscara de colisão que não seja retângulo será mais lento de resolver, mas quando se faz um jogo pequeno e simples, normalmente não é um problema. Note que você também pode importar sprites feito em outro software, clicando no botão Importar , no sprite e você também pode arrastar arquivos de imagem para o GameMaker IDE para adicioná-los como sprites ao Asset Browser.
